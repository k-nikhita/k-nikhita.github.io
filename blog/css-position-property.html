Differences Between Relative, Fixed & Absolute Positioning
"The best way to learn is to teach."
- Frank Oppenheimer


Hello! Welcome to my first technical blog. I chose this topic because I personally found it quite confusing when I first came across it. And this blog is my attempt to solidify my understanding. My hope is that in trying to do this, I might help someone who ends up reading this. So let's dive in.


The position property helps us specify the type of positioning method used for an element. This is important because depending on what we're trying to achieve on a webpage, we might want to position elements differently.

There are five different values that can be assigned to this property:

  • static
  • relative
  • fixed
  • absolute
  • sticky

In this article, I will primarily focus on the differences between relative, fixed and absolute positioning.


The first thing to note in general is that once you have set the position property for any element, they will then be positioned using the top, bottom, left, and right properties. And that these properties will not work unless the position property is set first. They will also work differently depending on the position value.

Before we delve into the differences between relative, fixed and absolute position, I would like to briefly discuss the static position. By default, all HTML elements are positioned as static, which means that they will always be positioned according to the normal flow of the page. For example:

This <h3> element has position: static;


To confirm this, you can right-click on the H3 element above and select "Inspect" to view the element's default position in DOM, as shown in the screenshot here.


position: relative;


An element with position: relative; is positioned relative to its normal position in the normal flow of the document.

Setting the top, right, bottom, and left properties of a relatively-positioned element will cause it to be adjusted away from its normal position. This will not adjust other content to fit into any gap left by the element after having moved from its former position.

If you right-click to inspect the image or screenshot showing the position of the h3 element in DOM, you will see that the position of this image is set to relative and that it is positioned to move 18px away from the top relative to its position in the normal flow of the document.

position: fixed;


Think of floating headers! Cookie consents! Newsletter pop-ups!


An element with position: fixed; is positioned relative to the viewport, i.e. the visible area of a web page. This means that it always stays in the same place even when the page is scrolled. Just like with the relative position, the top, right, bottom, and left properties are used to position the element.


This <div> element has position: fixed;
Can you spot the fixed element on this page? Again, you could right-click the <div> element to inspect the properties I used in CSS, but here is a screenshot of the CSS that I used:



A fixed element does not leave a gap in the page where it would have normally been located. This means that the rest of the content flows as if the element had never been there.


position: absolute;


An element with position: absolute; is positioned relative to its nearest positioned parent (as opposed to being positioned relative to the viewport in the case of fixed positioning).
However, if an absolute positioned element has no positioned parents, it uses the document body, and moves along with page scrolling. Please note that an element with the default, static position, does not count.
An absolutely positioned element is kind of ignored by its parent. It's on its own. It can be attached to the parent, yes, but the parent is not affected by it at all. You will see this in action in the example below.
