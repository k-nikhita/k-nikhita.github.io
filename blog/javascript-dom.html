Making sense of HTML, CSS & Javascript: The Anatomy of a Website


My favourite analogy to help make sense of the three web languages that form the building blocks of most websites is to liken them to a human body. A human body has different systems and functions that allow us to do what we do and how we interact with our surroundings.

As you can see in the image below, HTML (Hyper Text Markup Language) is like the skeleton of the human body. Like the skeleton, it defines the content and basic structure of a website - the bare bones. It holds everything together and provides a base to hang everything off. However, while an HTML only website has all of its body parts, it doesn't look like much without its accessories or personal style, nor is it capable of moving, speaking or interacting with its surroundings.

CSS (Cascading Style Sheet) is the website's clothing and accessories. It is responsible for outlining the colours, font and positioning of a website's contents, thereby lending it personal style. In the image below, you can tell that the girl is a dancer because of her clothing, dancing socks and headband (and her pose of course!). CSS is usually written in a separate file to HTML but linked to it. Without an HTML file to link to, CSS can't do anything as there is nothing to style, much like clothing and shoes are useless without a human body to dress.


HTML and CSS together can help us create well-structured, attractive websites but without Javascript, just like the muscles in a human body, your websites cannot interact with its users much like a mannequin in a store window. Javascript is a programming language that controls the behaviour of the website and can be compared to the human body's ability to perform actions such as walking or talking (and in the image below: dancing), thanks to its muscular (& nervous) system.



Control Flow


Control Flow can be defined as the order in which the computer executes statements in a script. In Javascript, the computer runs code from top to bottom, i.e. it starts from the first line and ends at the last line, unless it hits a statement or set of instructions (such as loops, conditions or functions) that affect the control flow of the program.

This can be likened to writing a script or set of instructions for any of the numerous activities that humans undertake every day. Let's take going for a walk for example. A basic set of steps involved in performing this activity might include:

  • Getting out of bed.
  • Picking what to wear (Since this will depend on the weather, an additional set of instructions based on conditions would apply. Such as:
    • If it is not raining, then put on a top and a pair of shorts.
    • If it is raining, then add a rain-jacket to your attire.
  • Put on walking shoes.
  • Grab the keys of the house.
  • Lock the house.
  • Once out of the house, start walking.

Now the steps outlined above are not exhaustive by any means and may vary for each individual depending on many factors such as their personal preferences, location, climate, etc. The point I am trying to make, however, is that there is a logical order to the steps that one would follow and in the case of computer, this order would be top to bottom.

Humans are creatures of habit. And most habits are automatic, hard to break and form a (repetitive) pattern of behaviour that responds to certain stimuli. Imagine that we were trying to a build a habit of going for a walk each morning for 31 days. In that case, we would end up repeating the above set of instructions, 31 times. This is where loops come in.

Loops in Javascript offer a quick and easy way to do something repeatedly. There are many different kinds of loops, but they all esentially do the same thing: they repeat an action (or set of actions), some number of times.



What the heck is DOM and why should I care?


First things first, DOM stands for Document Object Model.

Okay, so now what? By now, we know that a website is made up HTML elements, attributes, CSS styles, etc. that make up the structure and content of a website (or document in the web). Without a structured way of representing these building blocks and their relation to each other, one can imagine how messy things would get if you were trying to manipulate them.

This is where DOM comes in! The DOM in simple terms is a programming interface that defines the logical structure of web documents and the way they are accessed and manipulated. In the DOM, model, all of the properties, methods, and events available for manipulating and creating web pages are organized into objects; and the structure of these objects is represented in the form of a tree.



It is important to note that unlike HTML, CSS and Javascript, DOM is not a programming language but instead a web interface that allows us to access a document and its elements and even manipulate them. To use an analogy, you could see DOM as an interpreter between HTML and Javascript, in that it makes sense of the different languages and provides an interface for communication between someone who speaks the language and another that doesn't.

You can use DOM to for example, change a document's text colour, background colour, colour of hyperlinks, etc.

Functions


A Javascript function is a block of code that is written to perform a particular task. It can be likened to a procedure - a set of statements that perform a task. But for a procedure to qualify as a function, it should take some input and return an output, based on a some obvious (or predefined) relationship between the input and the output. So, to use a function, we need to define it somewhere in our code, where we establish this relationship between the input and output.

When writing a program, there might be several instances where we might want to repeat certain tasks without having to specify every single time what the code needs to do. This is where functions make life easier.
